The story

Students from the Web Technology course (2ID60) had to form groups by themselves. An online forum on Sakai was used by many students to find other teammates. This resulted in many posts on the forum from students who were still looking for a group. The forum on Sakai has two main disadvantages: firstly the lack of a clear overview and secondly the missing background information of students. As many students participate in university courses, you may want to collaborate with students you only know by face not by name.

We from group 8, Christian, Lars, Roy and Koen, thought: this process could be organized much more efficiently. That is why we have created Groupmolder: the easiest way to form groups.

Get Started

As a student, follow these steps:

  1. Sign up as a student or log in with Facebook
  2. Get a unique link from your teacher
  3. Choose your desired group members

As a teacher, follow these steps:

  1. Sign up as a teacher, Facebook not available
  2. Go to your project page
  3. Create a GroupMolder
  4. Share the link with your students

API documentation

Since we used MongoDB, all the accepted and returned information is in JSON.

The API is located under URL/api . All the given following links are relative to this URL.

Projects

To get all the projects use a GET request on /projects/ . The response contains an array with all the projects and corresponding information and groups with its members’ ID in it.

To get a specific project, use a GET request on /projects/”_id” . The “_id” part, is the “_id” attribute of the project in the database. When using the first request, the first attribute of an object given is the “_id”. The response is object which contains the project info and the groups with the members’ ID.

To create a project, use a POST method on /projects. The body must contain a “name”, “subjectCode”, “numStudents” and “numStudentsPerGroup”. The number of groups is automatically calculated and the number of needed groups is also created immediately.

To update a project, use a PUT method on /projects/”_id” . The body must contain a “name” and “subjectCode”. The response is the updated project.

To delete a project, use a DELETE method on /projects/”_id” . The response when completed is: {"result": "succes!"}.

Groups

To retrieve information about a group, use a GET method on /projects/”_id”/groups . The response is an array with all the groups of that project and all the member information.

To retrieve information about a specific group, use a GET method on /projects/”_id”/groups/”number” . The response is an object with the groupnumber and all the information about its members.

For the following 2 endpoints you have to be authenticated via the website (via Log in of Facebook login):

To check whether an user is allowed to join a group, use a GET method on /projects/”_id”/groups/”number”/check . The result is either {"error": "User already joined group"} or {"check": "passed"} or {"status": "User not logged in"}.

To join a group, use a POST method on /projects/”_id”/groups/”number” . You don’t have to provide a body, since you’re information is parsed from the user/_id header of your request. The response contains the specific information for that group.

User

The only method provided for a user is to get information about which groups the user joined. You also have to be authenticated by our website to do this query!

To get information about the projects that you joined, use a GET method on /users . The response is an array with all the information of a project that your joined.
